# 배포 가이드

## 기술 스택
- **프론트엔드**: React + Vite
- **백엔드**: Firebase (Auth, Firestore, Storage)
- **배포**: GitHub + Vercel

---

## 최초 배포 (처음 한 번만)

### 1. GitHub 레포 생성 및 푸시

```bash
# 프로젝트 폴더에서 실행
git init
git add .
git commit -m "Initial commit"

# GitHub CLI로 레포 생성 & 푸시
gh repo create workout-app --public --source=. --remote=origin --push
```

### 2. Firebase 환경변수 파일 생성 (`.env`)

프로젝트 루트에 `.env` 파일 생성:

```
VITE_FIREBASE_API_KEY=...
VITE_FIREBASE_AUTH_DOMAIN=...
VITE_FIREBASE_PROJECT_ID=...
VITE_FIREBASE_STORAGE_BUCKET=...
VITE_FIREBASE_MESSAGING_SENDER_ID=...
VITE_FIREBASE_APP_ID=...
VITE_FIREBASE_MEASUREMENT_ID=...
```

> `.env` 파일은 `.gitignore`에 포함되어 GitHub에 올라가지 않습니다.  
> Finder에서 숨김 파일 보기: **Cmd + Shift + .**

### 3. Vercel 배포

1. https://vercel.com 접속 → GitHub 계정으로 로그인
2. **Add New Project** → GitHub에서 `workout-app` 레포 선택
3. **Environment Variables** 섹션에 `.env` 파일 내용 붙여넣기
4. **Deploy** 클릭

### 4. Firebase 승인 도메인 추가

배포 후 구글 로그인 오류(`auth/unauthorized-domain`) 방지:

1. Firebase 콘솔 → **Authentication** → **Settings**
2. **승인된 도메인** → **도메인 추가**
3. Vercel 도메인 입력 (예: `workout-app-xxx.vercel.app`)

---

## 이후 업데이트 배포

코드 수정 후 GitHub에 푸시하면 Vercel이 자동으로 재배포합니다.

```bash
git add .
git commit -m "수정 내용 설명"
git push
```

> Vercel은 `main` 브랜치에 푸시될 때마다 자동 배포됩니다.

---

## 환경변수 수정이 필요한 경우

Vercel 대시보드 → 프로젝트 선택 → **Settings** → **Environment Variables**  
수정 후 **Deployments** → 최신 배포 → **Redeploy** 필요
